#### B2. 单元格大小和间距算法

**修改 `_HeatmapGrid.resizeEvent()`**:

当前逻辑（第 74-79 行）根据 `available / 53` 列计算 `_cell_step`，并限制为最小 10px。用根据实际可用宽度的新算法替换：

```python
def resizeEvent(self, event):
    super().resizeEvent(event)
    available_w = self.width() - _LEFT_MARGIN - _RIGHT_MARGIN
    # 53 列（一周一列，最多 53 周）
    # 目标：cell_size 在 16-22px 之间，gap 为 cell_size 的 15%，最大 4px
    ideal_step = available_w / 53
    self._cell_size = max(14, min(22, int(ideal_step * 0.85)))
    self._cell_gap = max(2, min(4, int(self._cell_size * 0.15)))
    self._cell_step = self._cell_size + self._cell_gap
    
    # 计算总高度：7 行 + 顶部/底部边距
    total_h = _TOP_MARGIN + 7 * self._cell_step + _BOTTOM_MARGIN + 20  # 20为图例空间
    self.setFixedHeight(total_h)  # 或根据模式使用 minimumHeight/maximumHeight
    self.update()
```

#### B7. 级别从 5 个增加到 7-8 个

**更改 `_level_for_count()`**:

当前使用 5 个级别（level_0 到 level_4），将数据等分为 25% 分位数。替换为使用 8 个级别（level_0 到 level_7），采用对数分桶以获得更好的视觉区分度：

```python
def _level_for_count(self, count: int, max_count: int) -> str:
    if count == 0:
        return "level_0"
    if max_count == 0:
        return "level_1"
    # 7 个非零级别的对数分桶
    ratio = count / max_count
    if ratio <= 0.01:  return "level_1"
    elif ratio <= 0.05: return "level_2"
    elif ratio <= 0.12: return "level_3"
    elif ratio <= 0.25: return "level_4"
    elif ratio <= 0.45: return "level_5"
    elif ratio <= 0.70: return "level_6"
    else:               return "level_7"
```

```python
def total_count(self) -> int:
    return sum(self._data.values())

def active_days(self) -> int:
    return sum(1 for v in self._data.values() if v > 0)

def longest_streak(self) -> int:
    """计算连续天数（count > 0 的连续天数最大长度）"""
    sorted_dates = sorted(self._data.keys())
    max_streak = current = 0
    for d in sorted_dates:
        if self._data[d] > 0:
            current += 1
            max_streak = max(max_streak, current)
        else:
            current = 0
    return max_streak

def monthly_count(self, year: int, month: int) -> int:
    return sum(c for d, c in self._data.items() if d.year == year and d.month == month)

def weekly_count(self, year: int, week: int) -> int:
    """给定 ISO 周的任务计数"""
    return sum(c for d, c in self._data.items() 
               if d.isocalendar()[0] == year and d.isocalendar()[1] == week)
```

### E. 颜色系统

#### E1. 向 DesignTokens 添加热力图级别

**当前**: `DesignTokens` 没有热力图字段，`AppConfig.heatmap_colors` 返回配置中的 dict。

**新方法**: 以编程方式从 `accent` 生成热力图颜色，而不是硬编码。这样做：
1. 向 `DesignTokens` 添加一个属性 `heatmap_gradient: list[str]`，返回 8 个十六进制颜色的列表。
2. 计算逻辑：以当前主题的 `accent` 为基准，生成从近白到全 accent 颜色的渐变。

**文件变更 -- `src/utils/design_tokens.py`**:

添加方法：
```python
@dataclass(frozen=True)
class DesignTokens:
    # ... 现有字段 ...
    
    def heatmap_gradient(self, levels: int = 8) -> list[str]:
        """从 background 颜色到 accent 颜色的渐变，生成 'levels' 个步骤。"""
        bg = QColor(self.bg_primary)
        accent = QColor(self.accent)
        colors = []
        for i in range(levels):
            ratio = i / (levels - 1)
            # 从 bg_primary 到 accent 的内插值
            r = int(bg.red() + (accent.red() - bg.red()) * ratio)
            g = int(bg.green() + (accent.green() - bg.green()) * ratio)
            b = int(bg.blue() + (accent.blue() - bg.blue()) * ratio)
            colors.append(f"#{r:02x}{g:02x}{b:02x}")
        return colors
```

Level 0（无任务）仍然使用 `bg_secondary` 或与背景略有不同的浅色，而不是渐变中的第一步。

实际层次结构：
- `level_0`: `bg_tertiary`（无任务的空单元格）
- `level_1` 到 `level_7`: 从 `bg_primary tint` 到 `accent` 的渐变

**当前**: `DEFAULT_CONFIG["display"]["heatmap_colors"]` 有 5 个硬编码的绿色级别。

**更改为**: 使用 8 个级别，使用主题生成的渐变。为了使配置在未来可能兼容，改变结构：

```python
    "custom_colors": {}  # 仅当 use_theme_gradient=False 时使用
}
```

对于向后兼容，检测旧的 5 级 dict 格式并迁移。

#### E3. 将渐变集成到 _HeatmapGrid

在 `_HeatmapGrid.__init__` 中，将 color dict 参数替换为对 `get_tokens().heatmap_gradient(levels)` 的调用：

```python
def __init__(self, model, levels=8, parent=None):
    ...
    self._levels = levels
    self._refresh_colors()
    
def _refresh_colors(self):
    tokens = get_tokens()
    self._colors = {
        "level_0": tokens.bg_tertiary,
    }
    gradient = tokens.heatmap_gradient(self._levels)
    for i in range(1, self._levels):
        self._colors[f"level_{i}"] = gradient[i - 1]
```

当主题改变（由于配置改变）时，`CalendarHeatmapWidget` 应该调用 `_main_grid._refresh_colors()` 和一个 `update()`。

#### E4. 主题兼容性

`heatmap_gradient()` 方法自动处理暗色/亮色兼容性，因为：
- 在暗色模式下，`bg_primary` 是 `#1a1b26`（深色），`accent` 是 `#7aa2f7`（亮蓝） -- 渐变从深色到蓝色
- 在亮色模式下，`bg_primary` 是 `#f5f4f0`（浅色），`accent` 是 `#5b8def`（深蓝） -- 渐变从浅色到蓝色

确保 `_draw_cell` 中的 `alpha = 40` 逻辑（未来日期）仍然适用于所有级别。

```python
def _on_date_selected(self, selected_date: date) -> None:
    if not self._guard_draft():
        return
    # 不再切换到页面 0 -- 我们已经在页面 0（分割模式）
    # 手动设置日期过滤器，无需切换页面
    self._filter_bar.blockSignals(True)
    self._filter_bar.reset()
    self._filter_bar.blockSignals(False)
    f = self._filter_bar.build_filter()
    f.date_from = selected_date
    f.date_to = selected_date
    self._filter_bar.filter_changed.emit(f)
```

6. 对于日期范围选择，添加一个新的处理程序：

```python
def _on_date_range_selected(self, d1: date, d2: date) -> None:
    if not self._guard_draft():
        return
    self._filter_bar.blockSignals(True)
    self._filter_bar.reset()
    self._filter_bar.blockSignals(False)
    f = self._filter_bar.build_filter()
    f.date_from = min(d1, d2)
    f.date_to = max(d1, d2)
    self._filter_bar.filter_changed.emit(f)
```

### I. 潜在挑战和缓解措施

1. **QSS 和自定义 QPainter 小部件**: QSS 不直接应用于 QPainter 绘制。所有热力图视觉样式必须在 `paintEvent()` 中通过 design_tokens 编程处理。QSS 只应用于包裹容器（`QWidget#calendarHeatmap`）和按钮/标签。

2. **拖选和右键菜单冲突**: 确保 `mousePressEvent` 区分左键（拖选开始）和右键（上下文菜单）。使用 `event.button()` 并相应处理。

3. **分割模式下热力图的高度**: 在分割模式下，热力图应占用约 220-280px 的总高度。使用 `setFixedHeight()` 或让 QSplitter 管理它。如果在 QVBoxLayout 内使用 QSplitter 用于（热力图 | 任务内容），热力图将具有可拖动的分割线。

4. **数据同步性能**: 每次任务更改时刷新整个热力图可能导致性能问题。如果延迟明显，考虑使用增量更新或节流刷新（每 500ms 最多一次）。为简单起见，直接从 HeatmapModel 重新加载数据是直接的，对于典型任务数量应该足够快。

5. **拖选时的工具提示**: 拖选时，应暂停工具提示以避免视觉混乱。使用 `_is_dragging` 标志跳过 `mouseMoveEvent` 中的工具提示更新。

6. **全屏模式下的导航栏**: 在全屏模式下，热力图应保留其导航栏（年份切换、跳转到今天），但 "collapsible panel" 标题栏应隐藏。

7. **主窗口内的 QSplitter**: 如果使用垂直 QSplitter 来划分热力图和任务内容，需要小心地与现有的水平 QSplitter（任务列表 | 编辑面板）在嵌套中配合。外部 QSplitter（垂直）包裹内部 QSplitter（水平）。

#### J2. _HeatmapGrid 拖选实现（关键部分）

```python
class _HeatmapGrid(QWidget):
    date_selected = Signal(date)
    date_range_selected = Signal(date, date)
    date_hovered = Signal(object)
    
    def __init__(self, ...):
        ...
        self._drag_start: date | None = None
        self._drag_end: date | None = None
        self._is_dragging = False
        self._tooltip: "HeatmapTooltip" | None = None  # 由父部件设置
    
    def mousePressEvent(self, event):
        if event.button() == Qt.LeftButton:
            d = self._date_at_pos(event.pos())
            if d is not None:
                self._drag_start = d
                self._drag_end = d
                self._is_dragging = True
        elif event.button() == Qt.RightButton:
            self._show_context_menu(event)
    
    def mouseMoveEvent(self, event):
        d = self._date_at_pos(event.pos())
        if self._is_dragging and d is not None:
            self._drag_end = d
            self.update()
        else:
            # 仅在非拖拽时更新悬停和工具提示
            if d != self._hovered_date:
                self._hovered_date = d
                self.date_hovered.emit(d)
                self.update()
    
    def mouseReleaseEvent(self, event):
        if event.button() == Qt.LeftButton and self._is_dragging:
            self._is_dragging = False
            if self._drag_start and self._drag_end:
                if self._drag_start == self._drag_end:
                    self.date_selected.emit(self._drag_start)
                else:
                    self.date_range_selected.emit(
                        min(self._drag_start, self._drag_end),
                        max(self._drag_start, self._drag_end)
                    )
            self._drag_start = self._drag_end = None
            self.update()
    
    def _draw_cell(self, p, col, row, cell_date, count, max_count):
        # ... 现有绘制代码 ...
        # 之后，检查拖拽范围高亮
        if self._is_dragging and self._drag_start and self._drag_end:
            d1, d2 = min(self._drag_start, self._drag_end), max(self._drag_start, self._drag_end)
            if d1 <= cell_date <= d2:
                overlay = QColor(get_tokens().accent)
                overlay.setAlpha(30)
                p.setPen(Qt.PenStyle.NoPen)
                p.setBrush(overlay)
                p.drawRoundedRect(rect, 2, 2)
```
